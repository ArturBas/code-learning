Flexbox Own Cheat Sheet

Command / Parameter

Execution

display: flex;

child elements of flex containers will change size and location in response to the size and position of their parent container

NOTE:
- all divs with the class container are flex containers
- if they have children, the children are flex items
- a div with the declaration display: flex; will remain block level â€” no other elements will appear on the same line as it

display: inline-flex;

allows us to create flex containers that are also inline elements

NOTE: if the parent element is too small, the flex items will shrink to accommodate the parent container's size

justify-content: value;

allows to space items along the main axis

justify-content: flex-start;

all items will be positioned in order, starting from the left of the parent container, with no extra space between or before them

justify-content: flex-end;

all items will be positioned in order, with the last item starting on the right side of the parent container, with no extra space between or after them

justify-content: center;

all items will be positioned in order, in the center of the parent container with no extra space before, between, or after them

justify-content: space-around;

items will be positioned with equal space before and after each item, resulting in double the space between elements

justify-content: space-between;

items will be positioned with equal space between them, but no extra space before the first or after the last elements

align-items: value;

allows to space items along the cross axis

align-items: flex-start;

all elements will be positioned at the top of the parent container

align-items: flex-end;

all elements will be positioned at the bottom of the parent container

align-items: center;

the center of all elements will be positioned halfway between the top and bottom of the parent container

align-items: baseline;

the bottom of the content of all items will be aligned with each other

align-items: stretch;

if possible, the items will stretch from top to bottom of the container (this is the default value

NOTE:
- elements with a specified height will not stretch
- elements with a minimum height or no height specified will stretch)

min-height: value;
max-height: value;
min-width: value;
max-width: value;

properties that ensure an element is at least a certain size or at most a certain size

NOTE:
- minimum and maximum widths will take precedence over flex-grow and flex-shrink
- margins are unaffected by flex-grow
- minimum and maximum widths will take precedence over flex-grow
- flex-grow will only be employed if the parent container is too large or the browser is adjusted
- flex-shrink will only be employed if the parent container is too small or the browser is adjusted

flex-grow: value;

allows us to specify if items should grow to fill a container and also which items should grow proportionally more or less than others

NOTE:
- flex items do not grow unless the flex-grow property is declared because the default value of flex-grow is 0
- margins are unaffected by flex-grow
- minimum and maximum widths will take precedence over flex-grow
- flex-grow will only be employed if the parent container is too large or the browser is adjusted

flex-shrink: value;

specifies which elements will shrink and in what proportions

NOTE:
- flex items shrinks automatically when the flex container is too small, even though we had not declared the property - this is because the default value of flex-shrink is 1
- margins are unaffected by flex-shrink
- minimum and maximum widths will take precedence over flex-shrink
- flex-shrink will only be employed if the parent container is too small or the browser is adjusted

flex-basis: value;

allows to specify the width of an item before it stretches or shrinks

flex: flex-grow flex-shrink flex-basis;

allows to declare flex-grow, flex-shrink, and flex-basis (in that order) all in one line

NOTE:
- the flex property is different from the flex value used for the display property
- there is possible to use the flex property to declare flex-grow and flex-shrink, and miss flex-basis
- there is possible to use the flex property to declare flex-grow and flex-basis, and miss flex-shrink
- there is no way to set only flex-shrink and flex-basis using 2 values

flex-wrap: wrap;

child elements of a flex container that don't fit into a row will move down to the next line

NOTE:
- the flex-wrap property is declared on flex containers

flex-wrap: wrap-reverse;

the same functionality as wrap, but the order of rows within a flex container is reversed (for example, in a 2-row flexbox, the first row from a wrap container will become the second in wrap-reverse and the second row from the wrap container will become the first in wrap-reverse)

NOTE:
- the flex-wrap property is declared on flex containers

flex-wrap: nowrap;

prevents items from wrapping; this is the default value and is only necessary to override a wrap value set by a different CSS rule

NOTE:
- the flex-wrap property is declared on flex containers

align-content: flex-start;

all rows of elements will be positioned at the top of the parent container with no extra space between

NOTE:
- the align-content property is declared on flex containers

align-content: flex-end;

all rows of elements will be positioned at the bottom of the parent container with no extra space between

NOTE:
- the align-content property is declared on flex containers

align-content: flex-center;

all rows of elements will be positioned at the center of the parent element with no extra space between

NOTE:
- the align-content property is declared on flex containers

align-content: space-between;

all rows of elements will be spaced evenly from the top to the bottom of the container with no space above the first or below the last

NOTE:
- the align-content property is declared on flex containers

align-content: space-around;

all rows of elements will be spaced evenly from the top to the bottom of the container with the same amount of space at the top and bottom and between each element

NOTE:
- the align-content property is declared on flex containers

align-content: stretch;

if a minimum height or no height is specified, the rows of elements will stretch to fill the parent container from top to bottom (default value)

NOTE:
- the align-content property is declared on flex containers

align-items: value;

allows for aligning elements within a single row and can take following values:
- baseline;
- center;
- flex-start;
- flex-end;
- stretch;

flex axes

flex containers have two axes:
- a main axis (by default, horizontal)
- a cross axis (vertical)
The main axis is used to position flex items with the following properties:
- justify-content
- flex-wrap
- flex-grow
- flex-shrink
The cross axis is used to position flex items with the following properties:
- align-items
- align-content
The main axis and cross axis are interchangeable. We can switch them using the flex-direction property.

flex-direction: row;

elements will be positioned from left to right across the parent element starting from the top left corner (default)

NOTE:
- the flex-direction property is declared on flex containers

flex-direction: row-reverse;

elements will be positioned from right to left across the parent element starting from the top right corner

NOTE:
- the flex-direction property is declared on flex containers

flex-direction: column;

elements will be positioned from top to bottom of the parent element starting from the top left corner

NOTE:
- the flex-direction property is declared on flex containers

flex-direction: column-reverse;

elements will be positioned from the bottom to the top of the parent element starting from the bottom left corner

NOTE:
- the flex-direction property is declared on flex containers

flex-flow: flex-direction flex-wrap;

allows to declare both the flex-direction and flex-wrap properties (in that order) in one line

NOTE:
- all values for flex-direction and flex-wrap are accepted
- the flex-flow property is declared on flex containers
