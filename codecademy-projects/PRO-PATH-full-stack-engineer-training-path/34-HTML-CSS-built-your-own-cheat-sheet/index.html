Windows PowerShell Own Cheat Sheet

Command / Parameter

Execution

gci

checking directory contents

sl

moving between directories

mkdir

creating new directory

explorer

opening Windows Explorer

rm

remove item

pushd

moving into the given directory after marking the current directory so it can be returned

popd

backing to the last directory you were in before using pushd

ni

creating new file

cp

copying file

Move-Item
-Path -Destination

moving file from given path to given -Destination

mv

renaming file

write

putting a content of one text (by using "") directly into the command-line window or into the file (by using > and the file name - then it will overwrote what was already there or by using >> and the file name - then it will append the new content)

gc

reading file

*

wildcard, can be used to replace any part of a file name

sls "unique existence" *.txt

spitting out all the lines containing string (which is put in "") from any file in our directory ending in .txt

control-c

stopping infinity loops or long tasks

Get-Help

showing the list of parameters possible to use for given cmdlet

Get-Help
-online

asking the browser to open a page on Microsoft's TechNet portal that explains all the parameters possible to use for given cmdlet

-exclude

omitting the specified items

measure

measuring specified properties such like line (-l), word (-w), character (-c) or (-IgnoreWhiteSpace)

|

piping, sort of like redirection, it tells it to take the output of one cmdlet and use it as the input for another

python

running Python scripts by following the command by the path for the script you want (itâ€™s easiest to first navigate to the directory with the script in it, and then just enter python script-name.py)
